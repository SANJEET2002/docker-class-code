Docker Volumes Demo (Bind Mounts)

This small demo shows how to run a Node.js container using the latest node:latest image and mount a local directory into the container using Docker bind mounts.

We will start the container with an interactive bash shell to see the mounted files inside the container.

ğŸš€ 2. Run Container With Bind Mount

Run the following command from the directory above your app folder:

docker run -it \
  -v $(pwd)/app:/usr/src/app \
  -w /usr/src/app \
  node:latest bash

Explanation
Flag	Meaning
-it	Interactive terminal
-v	Mount a local directory into container
-w	Set working directory inside container
node:latest	Base image
bash	Start bash shell

This is a bind mount, meaning changes are synced between host and container in real time.

Useful for local development without rebuilding images.

You can edit files locally and re-run commands inside container instantly.
